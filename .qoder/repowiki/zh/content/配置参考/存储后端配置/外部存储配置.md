## 简介
本文档详细说明了 Athens 当前支持的存储配置选项。**重要说明**：外部存储配置功能已在最新版本中被完全移除，不再支持自定义外部存储后端。目前 Athens 仅支持 MongoDB 作为存储后端。

## 当前支持的存储类型
根据当前代码实现，Athens 仅支持以下存储类型：

- **MongoDB** - 官方推荐的生产存储解决方案
- **内存存储** - 仅用于开发环境
- **磁盘存储** - 本地文件系统存储
- **Google Cloud Storage** - GCP 对象存储
- **AWS S3** - Amazon S3 兼容存储
- **Azure Blob Storage** - Microsoft Azure Blob 存储
- **Minio** - S3 兼容对象存储

**已移除的功能**：外部存储（External Storage）配置选项已被完全移除，不再可用。

## 配置参数与环境变量
### 基础配置
- **StorageType** - 存储类型，必须设置为支持的类型之一
- **Timeout** - 外部网络调用超时时间（秒）

### 环境变量映射
- `ATHENS_STORAGE_TYPE` - 覆盖 StorageType
- `ATHENS_TIMEOUT` - 覆盖 Timeout

## MongoDB 存储配置
### 配置参数
- **URL** - MongoDB 连接字符串（必需）
- **DefaultDBName** - 默认数据库名称，默认为 "athens"
- **DefaultCollectionName** - 默认集合名称，默认为 "modules"
- **CertPath** - SSL 证书路径
- **InsecureConn** - 是否允许不安全连接

### 环境变量
- `ATHENS_MONGO_STORAGE_URL` - MongoDB 连接 URL
- `ATHENS_MONGO_DEFAULT_DATABASE` - 默认数据库名称
- `ATHENS_MONGO_DEFAULT_COLLECTION` - 默认集合名称
- `ATHENS_MONGO_CERT_PATH` - 证书路径
- `ATHENS_MONGO_INSECURE` - 不安全连接标志

## 存储后端接口
### Backend 接口
存储后端通过统一的 Backend 接口抽象，该接口组合了以下四个基本操作：

```go
type Backend interface {
```

## 配置验证与加载
### 验证逻辑
当前配置验证逻辑仅支持 MongoDB 存储类型：

```go
func validateStorage(validate *validator.Validate, storageType string, config *Storage) error {
    if storageType != "mongo" {
        return fmt.Errorf("only 'mongo' storage type is supported, got: %q", storageType)
    }
    return validate.Struct(config.Mongo)
}
```

### 默认配置
默认存储类型为 "memory"，但生产环境强烈建议使用 "mongo"。

## 故障处理指南
### 常见问题
- **连接失败** - 检查 MongoDB 连接字符串和网络连通性
- **认证失败** - 验证用户名密码和权限设置
- **超时错误** - 调整 Timeout 配置和网络延迟

### 排查步骤
1. 检查 MongoDB 服务状态
2. 验证连接凭据
3. 查看应用日志
4. 测试网络连通性

## 结论
Athens 已简化其存储配置选项，专注于提供稳定可靠的存储解决方案。MongoDB 作为唯一支持的存储后端，为企业级部署提供了强大的功能和可靠性。外部存储配置功能的移除使系统更加简洁，减少了维护复杂性。
